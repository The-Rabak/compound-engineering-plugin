MCP Servers Reference

The current plugin bundles exactly 1 MCP server: context7. Earlier docs that mentioned a bundled Playwright MCP server were stale; browser automation now ships as the separate agent-browser skill.

Inventory note

The generated plugin metadata for version 4.5.0 exposes only context7 under mcpServers. Use agent-browser for web automation tasks.

Context7

Context7 gives Claude up-to-date framework and library documentation without leaving the terminal.

Tool Description
resolve-library-idFind the canonical Context7 identifier for a framework or package.
get-library-docsFetch current documentation for the selected library.
 
{
  "mcpServers": {
    "context7": {
      "type": "http",
      "url": "https://mcp.context7.com/mcp"
    }
  }
}

Browser automation via skill

The plugin still supports browser automation, but it now does so through the agent-browser skill instead of a bundled MCP server.

Component Description
agent-browserCLI-based browser automation using Vercel's agent-browser with snapshot refs for navigation, clicks, fills, and screenshots.
 
npm install -g agent-browser
agent-browser install

# Then invoke the skill in Claude
skill: agent-browser

Manual configuration

If Context7 does not auto-load, add it manually to .claude/settings.json:

{
  "mcpServers": {
    "context7": {
      "type": "http",
      "url": "https://mcp.context7.com/mcp"
    }
  }
}
Server Requirement
Context7Internet access to the hosted HTTP endpoint.
